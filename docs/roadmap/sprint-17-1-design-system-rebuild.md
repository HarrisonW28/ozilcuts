# Sprint 17.1 — Design System Rebuild

## Goals
Rebuild the frontend visual foundation to establish a premium, modern barber product aesthetic.

## Product Direction
The UI should feel:
- premium
- calm
- minimal
- editorial
- touch-first
- native-quality

Inspired by:
- Linear
- Stripe
- Apple
- Cron
- Notion Calendar

## Focus Areas
- typography system
- spacing system
- radius consistency
- shadow hierarchy
- iconography
- colour palette
- reusable UI primitives
- interaction consistency

## Frontend Tasks
- rebuild buttons
- rebuild cards
- rebuild forms
- rebuild dropdowns
- rebuild modals/sheets
- standardize spacing tokens
- standardize shadows/radius
- improve component states

## UX Requirements
- reduce generic SaaS/admin feel
- reduce visual clutter
- improve visual hierarchy
- improve readability
- improve touch ergonomics

## Constraints
- preserve architecture
- no backend rewrites
- maintain reusable abstractions
- preserve responsive behavior

## QA Checklist
- responsive QA
- dark mode QA
- accessibility QA
- component consistency QA
- tablet QA

## Acceptance Criteria
- cohesive visual identity established
- cleaner and more premium UI
- improved consistency across app
