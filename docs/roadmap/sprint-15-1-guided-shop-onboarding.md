# Sprint 15.1 — Guided Shop Onboarding

## Goals
Allow non-technical barber shops to onboard without confusion or manual setup complexity.

## Product Direction
The onboarding experience should feel:
- guided
- calm
- modern
- opinionated
- extremely simple

Inspired by:
- Shopify onboarding
- Linear onboarding
- iPhone setup flow

## Setup Steps
1. Shop Details
2. Business Hours
3. Add Barbers
4. Add Services
5. Payments
6. Go Live

## Focus Areas
- onboarding wizard
- progress indicators
- setup persistence
- opinionated defaults
- friction reduction
- setup validation
- success states

## Frontend Tasks
- create onboarding flow layout
- implement multi-step wizard
- add progress tracker
- implement step validation
- create mobile-first onboarding UX
- add friendly helper text
- implement success/completion screens

## Backend Tasks
- onboarding state persistence
- shop setup defaults
- starter services/templates
- setup completion tracking

## UX Requirements
- avoid overwhelming users
- avoid giant forms
- avoid settings-heavy UX
- mobile-first onboarding
- reduce cognitive load

## Suggested Defaults
Services:
- Haircut
- Skin Fade
- Hair + Beard
- Beard Trim

Payments:
- Cash Only
- Deposits Enabled
- Tap To Pay Later

## QA Checklist
- mobile onboarding QA
- non-technical user QA
- validation QA
- progress persistence QA
- accessibility QA

## Acceptance Criteria
- non-technical shops can onboard successfully
- setup process feels simple and modern
- reduced setup friction
- reduced onboarding abandonment
